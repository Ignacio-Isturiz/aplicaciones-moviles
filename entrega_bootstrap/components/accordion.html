1. Acordeón Básico (Default)

El acordeón estándar permite alternar la visibilidad de bloques de contenido. Por defecto, solo un elemento puede estar abierto a la vez.

Estética Visual: Observa que este acordeón tiene una apariencia de tarjeta o caja contenedora. Tiene bordes definidos, esquinas redondeadas y una sombra elegante que lo separa del fondo.

Ejemplo Práctico: Ideal para una sección de Preguntas Frecuentes (FAQ) donde quieres que el contenido destaque como un bloque independiente.

2. Acordeón "Flush"

La variante .accordion-flush elimina el color de fondo predeterminado, los bordes y las esquinas redondeadas para integrarse mejor en contenedores de ancho completo.

Estética Visual: A diferencia del básico, el Flush elimina los bordes exteriores y las esquinas redondeadas. Se siente mucho más plano y se "funde" con el fondo.

Ejemplo Práctico: Perfecto para menús laterales o dashboards donde quieres una lista de colapsables que parezca texto puro integrado en la interfaz, sin el ruido visual de bordes adicionales.

3. Paneles Independientes (Always Open)

Al omitir el atributo data-bs-parent, cada sección actúa de forma independiente, permitiendo múltiples paneles abiertos simultáneamente.

Función: Permite al usuario mantener varias secciones abiertas al mismo tiempo.

Ejemplo Práctico: Útil en listas de filtros de búsqueda (ej. Filtro por precio, marca y color en un e-commerce) donde el usuario necesita ver y ajustar varias categorías a la vez.

En este caso, abrir el segundo panel no cerrará el primero. Esto se logra simplemente no especificando un padre común en el HTML.
